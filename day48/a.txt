컴포넌트 기반의 구조

컴포넌트 = 자바스크립트 함수 안에 html 코드를 작성
JSX : 자바스크립트 안에 html 코드가 혼재되어 나타내는 부분




컨텐츠 업데이트 시 트리 구조가 복잡해질수록 페이지 로딩과 리프레시가 느려짐

리액트는 virtual dom이라는 개념을 활용해서 업데이트할 컨텐츠를 모았다가 한번에 업데이트하는 특징이 있다.
중간 업데이트를 가상 dom으로 업데이트하되 실제 컨텐츠는 변함이 없고, 변경사항 설정이 완료되면 그때 업데이트한다.




Props -> 부모가 자식 컴포넌트에 객체 형태로 값을 전달할 수 있는데
         이 객체를 Props 라고 함



REACT의 주요개념

JSX - 자바스크립트 + html
      대부분 자바스크립트 먹는데 객체 형태의 데이터 전송시 주의
component - 각 기능별로 (함수) 파일을 따로 빼서 작업
            작업이 끝나면 export해서 작업함수가 외부로 나갈 수 있도록 할 것
Props - 부모 컴포넌트에서 자식 컴포넌트로 데이터, 값, 함수 모두 전달 가능
        상속 받을 자식 컴포넌트 인수에 Props든 함수 자체이든 인풋이 받아질 수 있게



동적 컴포넌트 - 사용자의 액션 등에 의해 값이 변하는 컴포넌트를 의미
               동적 컴포넌트를 위해 state 기능이 존재

               state란 '상태'를 의미하며 state 결과에 따라 다른 컴포넌트 결과를 렌더링한다.
